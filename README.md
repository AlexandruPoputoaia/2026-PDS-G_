# 2026-PDS-G

Skin lesion classification on the **PAD-UFES-20** dataset, built for the
ITU course *Projects in Data Science* (Spring 2026).

A Random Forest is trained on **55 hand-crafted image features**
covering shape, colour, texture, border irregularity, asymmetry,
diameter, and hair to distinguish **cancer** (BCC, MEL, SCC) from
**non-cancer** (ACK, NEV, SEK) lesions. All train/test splits and
cross-validation folds are grouped by `patient_id` to prevent
patient-level data leakage (Sasse et al., 2025).

## How to reproduce

```bash
# 1. Extract features from images → writes data/features.csv
python -m src.extract_features

# 2. CV comparison, hyperparameter tuning, final training → writes
#    model, predictions, cv_results.json, and validation-curve figures
python main.py

# 3. Generate the results dashboard
python -m src.plot_results

# 4. Run the open-question hair-feature ablation experiment
python -m src.open_question
```

To evaluate the saved model on a new dataset
update the image/mask paths in `src/extract_features.py`, re-run
step 1 to regenerate `features.csv`, then set `load_model = True` at
the bottom of `main.py` and re-run step 2.

## Methodology

- **Patient-grouped splits.** `GroupShuffleSplit` for dev/test and
  `GroupKFold(n_splits=5)` for cross-validation, both keyed on
  `patient_id`. Patients never span the train/test boundary even though
  many have multiple images.
- **No preprocessing leakage.** `StandardScaler` lives inside the
  scikit-learn `Pipeline`, fitted only on training folds.
- **Balanced accuracy** as the selection metric raw accuracy
  over-rewards predicting the majority class on this imbalanced dataset.
- **Three classifiers compared** in CV (Decision Tree, Random Forest,
  KNN) before selecting the winner.
- **`RandomizedSearchCV`** tunes the winning classifier (Random Forest)
  using the same group-aware folds.
- **`class_weight='balanced'`** on the Random Forest to further
  counteract class imbalance.
- **Binary over 6-class.** An empirical comparison on the development
  set confirmed that the binary formulation (Cancer vs Non-Cancer)
  outperforms 6-class classification in balanced accuracy (0.731 vs
  0.414), and is more clinically relevant as a triage signal.

## Open question: do hair features help?

`src/open_question.py` tests whether `hair_coverage` and
`hair_in_lesion` provide genuine signal or exploit a clinical-workflow
shortcut. Dermatologists often shave the skin around lesions they
consider suspicious before photographing, so less hair can spuriously
correlate with malignancy

The same tuned Random Forest is trained twice on identical patient-
grouped folds once with all 55 features, once with the two hair
features removed and balanced accuracy and AUC are compared.

**Result:** removing hair features changes balanced accuracy by only
+0.001 and AUC by 0.000. The model is not relying on a hair shortcut;
hair coverage is redundant with the signal already captured by colour
and texture features.

Output: `results/figures/open_question_hair.png`,
`results/open_question_results.json`.
